# 📄 ملخص مشروع: تطبيق مسابقة عائلية تفاعلي (ويب)

---

## 🎯 الهدف العام
بناء تطبيق ويب تفاعلي للمسابقات العائلية، يدعم **نوعين من اللعب** (فردي وجماعي)، مع **لوحة تحكم سرية** للمقدم، وتفاعل لحظي عبر الجوال أو الشاشة الرئيسية.

---

## 🧩 المكونات الرئيسية للتطبيق

### 1️⃣ شاشة العرض الرئيسية (المقدمة للجمهور)
- تعرض السؤال الحالي + الخيارات (في الوضع الجماعي).
- تعرض الباركود ورقم الغرفة (في الوضع الفردي).
- تعرض التايمر العد التنازلي.
- تعرض الإجابة الصحيحة (صح/خطأ) بعد الكشف.
- تعرض لوحة النتائج والترتيب.

### 2️⃣ لوحة التحكم السرية (خاصة بالمقدم)
- دخول برقم سري ثابت.
- اختيار نوع المسابقة: **فردي** أو **جماعي**.
- إنشاء غرفة جديدة (توليد باركود ورقم غرفة عشوائي).
- عرض قائمة اللاعبين المسجلين (في الفردي).
- التحكم بالأسئلة: اختيار السؤال، بدء التايمر، كشف الإجابة.
- إضافة النقاط يدوياً (في الجماعي) أو تلقائياً (في الفردي).
- عرض وتحديث لوحة النتائج.
- ضبط مدة التايمر (مثلاً ٢٠، ٣٠، ٤٥ ثانية).

### 3️⃣ واجهة المشارك (على جواله)
- في **الوضع الفردي**:
  - يمسح باركود الشاشة أو يدخل رقم الغرفة يدوياً.
  - يسجل اسمه ويختار لونه (أو لون عشوائي).
  - ينتظر بدء السؤال.
  - تظهر له الخيارات (٣ أو ٤) على شكل أزرار.
  - يختار خياراً، ثم يظهر له "انتظار".
  - بعد كشف الإجابة، يظهر له مربع أخضر ✅ أو أحمر ❌.
  - يرى نقاطه وتحديث الترتيب.
- في **الوضع الجماعي**:
  - لا يحتاج تسجيل، كل شيء يظهر على الشاشة الرئيسية فقط.
  - يشارك شفهياً مع فريقه.

## ⚙️ المتطلبات التقنية

### الجهة الخلفية (Backend)
- لغة: **PHP** أو **Node.js** أو **Python (Django/Flask)**.
- قاعدة بيانات: **MySQL** أو **SQLite** (لحفظ الأسئلة، اللاعبين، النقاط، الغرف).
- نظام **WebSockets (مثل Socket.io)** للتحديث اللحظي (ضروري جداً).
- توليد باركود (مكتبة مثل `qrcode`).

### الجهة الأمامية (Frontend)
- **HTML5 + CSS3 + JavaScript**.
- إطار عمل: **Vue.js** أو **React** أو حتى **vanilla JS** مع Socket.io.
- تصميم متجاوب مع الجوال (Mobile First).
- ألوان واضحة وأزرار كبيرة.

### البنية الأساسية (قاعدة البيانات)
- جدول `users` (اللاعبين): id, name, color, room_id, score.
- جدول `rooms` (الغرف): id, room_code, type (فردي/جماعي), status.
- جدول `questions`: id, question, option1, option2, option3, option4, correct_answer, difficulty.
- جدول `game_sessions`: id, room_id, question_id, start_time, end_time.

## 📋 سيناريو اللعب خطوة بخطوة

### الوضع الفردي
1. المقدم يفتح لوحة التحكم، يختار "فردي"، يضغط "إنشاء غرفة".
2. تظهر الشاشة الرئيسية باركود ورقم الغرفة.
3. اللاعبون يمسحون الباركود، يسجلون أسمائهم وألوانهم.
4. المقدم يرى قائمة اللاعبين، يختار سؤالاً، يضبط التايمر، يضغط "بدء".
5. كل لاعب يشوف السؤال والخيارات على جواله، يختار أحدها.
6. بعد انتهاء التايمر، المقدم يضغط "كشف الإجابة".
7. تظهر لكل لاعب إجابته (صح ✅ / خطأ ❌) وتُحدث النقاط تلقائياً.
8. تظهر لوحة النتائج على الشاشة الرئيسية.

### الوضع الجماعي
1. المقدم يختار "جماعي"، لا حاجة لباركود أو تسجيل.
2. تظهر الشاشة الرئيسية السؤال والخيارات، والفرق تناقش شفهياً.
3. المقدم يضغط "كشف الإجابة" بعد انتهاء الوقت.
4. يضيف النقاط يدوياً للفريق الفائز من لوحة التحكم.
5. تحديث لوحة النتائج يدوياً.
