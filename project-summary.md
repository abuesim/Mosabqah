# 📄 دليل مشروع: مُسَابَقَة عصومي (الاصدار 2.5.0)

---

## 🎯 الهدف العام
بناء تطبيق ويب تفاعلي للمسابقات والتحديات العائلية والاجتماعية بالوقت الفعلي، يدعم **نوعين من اللعب** (فردي وجماعي)، مع **لوحة تحكم سرية** للمقدم وجهاز **ريموت محمول للجوال** للتحكم عن بعد، وتفاعل لحظي عبر الجوال أو الشاشة الرئيسية للجمهور، بالإضافة إلى ميزات رصد النتائج، المنصات الختامية (الفردية والجماعية للفرق)، وسائل المساعدة المتطورة للمتسابقين، والاستيراد الجماعي للأسئلة من ملفات Excel.

## 🧩 المكونات الرئيسية للتطبيق

### 1️⃣ شاشة العرض الرئيسية (المقدمة للجمهور - TV)
- تعرض تعليمات الانضمام مع كود الغرفة والباركود (مُولد ديناميكياً لتوجيه اللاعبين للرابط الصحيح).
- تعرض قائمة اللاعبين المتصلين لحظياً بألوانهم وحالات اتصالهم.
- تعرض **شاشة الاستعداد والعد التنازلي التمهيدي** (من 5 إلى 1 ثانية) قبل عرض كل سؤال لبناء الحماس والإثارة.
- تعرض السؤال الحالي والخيارات الأربعة في شبكة عرض كبيرة وواضحة.
- **التحكم باللمس أو النقر المباشر**: يمكن للمقدم النقر مباشرة على خيار الإجابة في شاشة العرض (عند تفعيل وضع التحكم) ليقوم النظام تلقائياً بالتحقق وحساب النقاط للفريق وتغيير الأدوار.
- **التكيف التلقائي مع عدد الخيارات**: في حال كان السؤال يحتوي على خيارين فقط (مثل صح/خطأ) أو 3 خيارات، تختفي تلقائياً المربعات الفارغة من شاشة الجمهور لتبدو نظيفة وممتازة.
- تعرض التايمر البصري والعد التنازلي، وتصدر تكات صوتية مدمجة.
- تعرض إحصائيات إجابات اللاعبين كشكل بياني موزع (أعمدة نسبية) لتوضيح الخيارات المختارة من الجمهور قبل كشف النتيجة.
- **لوحة النتائج النهائية المدمجة**: تعرض منصة التتويج والنتائج الختامية بنظام متوازٍ وجذاب:
  - **الإنجاز الفردي**: الترتيب العام لأعلى المتسابقين نقاطاً.
  - **الإنجاز الجماعي**: الترتيب التراكمي لنقاط الفرق مصنفة حسب ألوانها التنافسية (أحمر، أخضر، أزرق، أصفر) مع إحصائية لعدد المتسابقين في كل فريق.

### 2️⃣ لوحة التحكم السرية (خاصة بالمقدم - Admin Web)
- محمية برمز سري (`ADMIN_PASSWORD`).
- إمكانية تحديد **اسم مخصص للمسابقة** (مثل: تحدي الأربعاء) وتعيين **رمز غرفة ثابت رقمي** (مثل: 1234) يدعم إعادة التهيئة والتصفير الآلي للمشاركين عند إعادة الاستخدام لبدء مسابقة جديدة بالرمز نفسه.
- اختيار نوع اللعب عبر **أزرار وكروت تفاعلية أنيقة** بدلاً من القوائم التقليدية:
  - **لعب فردي**: يتفاعل اللاعبون بالهواتف ويحصلون على النقاط تلقائياً حسب صحة الإجابة وسرعتها (100 نقطة أساسية + حتى 50 نقطة إضافية كمكافأة سرعة = 150 نقطة كحد أقصى للسؤال).
  - **لعب جماعي**: لعب شفهي يتم رصده وإدارة نقاط الفرق يدوياً من لوحة التحكم، مع **إمكانية اختيار عدد الفرق المشاركة (2 أو 3 أو 4 فرق)** لتخصيص اللعبة تلقائياً.
- التحكم بالتايمر (20، 30، 45، 60 ثانية).
- إحصاء عدد الإجابات المستلمة لحظياً لكل سؤال مطروح.
- **بنك الأسئلة**: عرض الأسئلة المتوفرة مع إمكانية طرح أي سؤال بضغطة زر، تصفية الأسئلة ديناميكياً بحسب **التصنيف** (عام، إسلامي، ألغاز، علوم) وبحسب **مستوى الصعوبة** (سهل 🟢، متوسط 🟡، صعب 🔴)، وكشف الإجابات وحذف الأسئلة نهائياً.
- **أدوات تعبئة الأسئلة**:
  - إضافة سؤال جديد يدوياً عبر نموذج إدخال سريع ونظيف.
  - **تحميل قالب Excel**: تنزيل قالب CSV منسق برمز UTF-8 BOM لضمان عمله في Excel باللغة العربية دون مشاكل في الحروف.
  - **رفع أسئلة من Excel**: استيراد مئات الأسئلة دفعة واحدة من الملف وتحديث البنك فوراً.
- رصد وتعديل النقاط يدوياً ونقل المتسابقين بين الفرق وسحبهم مع مزامنة فورية للألوان والنقاط.
- **تصدير النتائج**: تصدير قائمة الترتيب واللاعبين كاملة بصيغة ملف CSV متوافق مع Excel.
- **إظهار وإخفاء الترتيب**: زر مخصص لبث قائمة الترتيب الحالي كغطاء زجاجي (Overlay) على شاشات هواتف المتسابقين، مع ميزة الإخفاء التلقائي بعد 8 ثوانٍ.

### 3️⃣ ريموت التحكم المحمول للجوال (`remote.html` - Mobile Remote)
- واجهة خاصة مهيأة تماماً لشاشات الهواتف الذكية تمنح المقدم حرية كاملة للتنقل في القاعة وإدارة المسابقة.
- تمكنه من: طرح سؤال عشوائي، إرسال سؤال تجريبي، كشف الإجابة، إنهاء المسابقة، وتعديل نقاط المتسابقين يدوياً بلمسة واحدة.
- استعراض بنك الأسئلة وفلترتها بالصعوبة والتصنيف وطرحها مباشرة.
- تتبع حالة السؤال النشط والعداد التنازلي وعدد الإجابات المستلمة.
- زر مستقل للتحكم بإظهار وإخفاء الترتيب على جوالات المتسابقين.

### 4️⃣ واجهة المشارك (على جواله - Player Page)
- **تدفق انضمام ثنائي الخطوات**: يقوم اللاعب بكتابة رمز الغرفة (رقم المسابقة) أولاً للتأكد من وجود الغرفة على السيرفر وعرض اسم المسابقة النشط، ثم يُكشف له النموذج الفرعي لكتابة اسمه واختيار لونه المفضل، مما يمنع انضمام اللاعبين لغرف غير موجودة أو خاطئة.
- **تأمين الجلسة واستعادة النقاط**: يتم إنشاء معرّف رقمي فريد ومستمر (`playerId`) في ذاكرة المتصفح `localStorage`؛ لو قفل شاشته أو حدثت الصفحة، يعود فوراً لكامل نقاطه السابقة دون مشاكل.
- يظهر اسمه بوضوح في الشريط العلوي (الـ HUD Navbar) طوال فترة اللعب.
- يظهر له بادج ذهبي متوهج يعرض نقاطه الحالية (`⭐ X نقطة`).
- تظهر له شاشة الاستعداد ("استعد للسؤال التالي...") مع عداد تنازلي بصري وصوتي متزامن تماماً مع شاشة التلفزيون.
- تظهر له خيارات الإجابة الأربعة كأزرار لمسية ملونة ضخمة وسهلة الاستخدام (مع إخفاء الأزرار للخيارات الفارغة تلقائياً).
- تطلق اهتزازات تفاعلية ومؤثرات صوتية خاصة بالصح والخطأ عند كشف النتيجة.
- **وسائل المساعدة المتطورة (Lifelines)**:
  - **حذف إجابتين (✂️ 50/50)**: إخفاء خيارين خاطئين من الخيارات المتاحة (حد أقصى مرتين باللعبة).
  - **تمديد الوقت (⏱️ +20ث)**: تمديد مؤقت الإجابة للسؤال الحالي بمقدار 20 ثانية إضافية لجميع متسابقي الغرفة وشاشات العرض والريموت (حد أقصى مرتين باللعبة لكل لاعب).
- **لوحة الترتيب العائمة**: تظهر تلقائياً كـ Glassmorphism Overlay لعرض ترتيبه الحالي وتختفي ذاتياً بعد 8 ثوانٍ أو عند إخفائها من المقدم.
- **شاشة النهاية**: تعرض مجموع نقاطه النهائي، بالإضافة إلى **المركز والترتيب الدقيق** (مثل: المركز الأول 🏆🥇)، مع إظهار لوحة مخصصة للتعادل **("مكرر مع: [الأسماء]")** في حال تعادله مع متنافسين آخرين.

## ⚙️ المتطلبات التقنية والبنية الأساسية

### خادم التطبيق (Backend)
- **Node.js + Express**: خادم ويب خفيف وسريع لمعالجة وخدمة الملفات الثابتة.
- **Socket.io**: لتزامن البيانات بالوقت الفعلي وثنائي الاتجاه بين المقدم والجمهور واللاعبين.
- **SQLite3 / @libsql/client**: قاعدة بيانات مدمجة لحفظ الأسئلة، اللاعبين، النقاط، والغرف دون الحاجة لسيرفر خارجي (مهيأة للحفظ الدائم).
- **Web Audio API**: لتوليد المؤثرات الصوتية برمجياً في المتصفح، مما يضمن عمل الأصوات فوراً دون الحاجة لتحميل ملفات صوتية خارجية ضخمة.

### بنية قاعدة البيانات (Database Schema)
- جدول `rooms`: id, type, status, current_question_id, question_status, timer_duration, question_start_time, show_scoreboard.
- جدول `users` (اللاعبين والفرق): id, room_id, name, color, score, is_active, team_id, streak, lifelines_remaining, lifelines_time_remaining.
- جدول `questions`: id, question_text, option1, option2, option3, option4, correct_option, difficulty, category, image_url.
- جدول `player_answers`: id, room_id, player_id, question_id, chosen_option, is_correct, answered_in_seconds.

## 🚀 تعليمات التشغيل والنشر

### التشغيل المحلي:
1. قم بتثبيت الحزم البرمجية:
   ```bash
   npm install
   ```
2. ابدأ تشغيل الخادم:
   ```bash
   npm start
   ```
3. افتح الرابط في متصفحك: `http://localhost:3005`

### النشر على الإنترنت (Render / Docker):
المشروع مجهز بالكامل للرفع على GitHub والربط المباشر مع منصات الاستضافة السحابية مثل **Render.com** كـ **Web Service** تعتمد على **Docker**؛ حيث سيقوم الخادم ببناء وتشغيل التطبيق أوتوماتيكياً وقراءة كلمة المرور الخاصة بلوحة التحكم من متغير البيئة `ADMIN_PASSWORD` المرفق في لوحة Render بأمان تام.
