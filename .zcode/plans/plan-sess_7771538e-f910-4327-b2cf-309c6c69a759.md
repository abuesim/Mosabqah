# TOP 10 — دعم الجولات المتعددة

إضافة نظام جولات متعددة للعبة TOP 10: المقدم يختار عدد الجولات (1-100)، ثم يملأ كل جولة (عشوائي/محدد/مخصص). ينتقل بين الجولات يدوياً بزر "الجولة التالية".

## 🏗️ الاستراتيجية: "المرآة النشطة" (الخيار الآمن)

بدل ما أغيّر بنية البيانات الحالية (اللي تقرأها 6 أماكن حسّاسة)، أبقي `top10Items` و `top10Prompt` كـ **نسخة الجولة الحالية**، وأضيف مصفوفة `top10Rounds[]` منفصلة فيها كل الجولات المجدولة.

```
top10Rounds[]          ← كل الجولات (الجدولة الكاملة)
  ↓ عند بدء جولة
top10Prompt            ← ينسخ الجولة الحالية هنا
top10Items             ← ينسخ الجولة الحالية هنا (مع revealed: false)
  ↓ يقرأها
top10-answer/route.ts  ← ما يتغير أبداً ✓
tv/page.tsx            ← ما يتغير أبداً ✓
player/page.tsx        ← ما يتغير أبداً ✓
timeout/route.ts       ← ما يتغير أبداً ✓
```

**النتيجة:** API الإجابات + TV + اللاعبين + timeout كلها تبقى زي ما هي — لا خطر كسر اللعبة.

## 1️⃣ نموذج البيانات — `src/lib/db.ts`

إضافة حقلين جديدين على `Session` (الحقول القديمة تبقى كمرآة الجولة النشطة):

```typescript
top10Rounds?: Array<{
  id: string;              // "round-1", "round-2"...
  prompt: string;
  bankQuestionId: string | null;
  selectionMode: "random" | "custom" | "selected";
  items: Array<{           // قالب نظيف (بدون foundById)
    id: string; answer: string; aliases: string[]; points: number;
  }>;
  status?: "pending" | "active" | "completed";
}>;
top10CurrentRoundIndex?: number;   // مؤشر الجولة الحالية (0-based)
```

`firestore.rules` — لا تغيير (الحقول داخل session doc، والقواعد field-agnostic).

## 2️⃣ معالج الإعداد — `src/app/dashboard/games/page.tsx` + `Top10BankPicker.tsx`

### عدّاد الجولات
في أعلى المرحلة 2، أضيف عدّاد (1-100) بأزرار +/- وحقل رقمي. افتراضي = 1.

### إدارة مسودات الجولات
حالة جديدة في games/page.tsx:
- `top10RoundDrafts: Top10RoundDraft[]` — مصفوفة مسودات (طولها = عدد الجولات)
- `top10ActiveRoundIdx: number` — أي جولة يعدّلها المستخدم حالياً

عند تغيير عدّاد الجولات، تُمدد/تُقصّ المصفوفة تلقائياً (الجولات الجديدة تبدأ فارغة = مخصص).

### واجهة الجولة النشطة
شريط تنقّل فوق Top10BankPicker:
- **« الجولة السابقة | الجولة 2 من 5 | الجولة التالية »**
- مؤشر بصري: الجولات المكتملة ✓، الحالية مظللة، الباقي رمادي

`Top10BankPicker` نفسه **يبقى كما هو تقريباً** — بس بدل ما يقرأ state واحد، يقرأ/يكتب المسودة النشطة فقط. أعدّل props لقبول/إرجاع مسودة محددة:

```typescript
// Top10BankPicker — props معدّلة
mode, onModeChange, selectedId, onChooseQuestion,
prompt, onPromptChange, entries, onEntriesChange,
// تبقى زي ما هي — بس تستبدل بمسودة الجولة النشطة
```

### إنشاء الجلسة (createSession)
بدل ما يحفظ سؤال واحد، يحفظ:
```typescript
top10Rounds: top10RoundDrafts.map((draft, i) => ({
  id: `round-${i+1}`,
  prompt: draft.prompt,
  bankQuestionId: draft.mode === "custom" ? null : draft.selectedId,
  selectionMode: draft.mode,
  items: draft.entries.map((entry, j) => ({...})),
  status: "pending",
})),
top10CurrentRoundIndex: 0,
// الجولة الأولى تُنسخ للحقول النشطة فوراً:
top10Prompt: drafts[0].prompt,
top10Items: drafts[0].items.map(...revealed:false),
```

### التحقق
كل جولة لازم لها: prompt + 10 إجابات مختلفة. لو أي جولة ناقصة → خطأ يحدد رقم الجولة.

## 3️⃣ وحدة تحكم المقدم — `src/app/dashboard/sessions/page.tsx`

### مؤشر تقدم الجولة
في لوحة تحكم TOP 10، أضف شريط: **"الجولة 2 من 5"** مع نقاط بصرية للجولات (مكتملة ✓ / نشطة ● / معلّقة ○).

### بدء الجولة الأولى
`handleStartTop10Round` الحالية تبقى (تنسخ `top10Items` مع `revealed:false`). أضيف قبلها: نسخ `top10Rounds[0]` → الحقول النشطة + تعيين `status:"active"`.

### زر "الجولة التالية" (جديد)
عند `questionStatus === "revealed"`:
1. علم `top10Rounds[currentIndex].status = "completed"`
2. `top10CurrentRoundIndex += 1`
3. لو في جولات بعدها: انسخ الجولة التالية → `top10Prompt` + `top10Items` (revealed:false), عيّن `questionStatus:"showing"`, `questionStartedAt: +3s`
4. لو آخر جولة: اعرض "اكتملت جميع الجولات" (المقدم ينهي اللعبة عادي)

### تعديل الإعدادات أثناء اللعب
`handleSaveTop10Settings` الحالية تبقى للجولة النشطة فقط. ما يُسمح بتعديل الجولات القادمة أثناء اللعب (مثل قيد baathra).
