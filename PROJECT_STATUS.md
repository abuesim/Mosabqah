# توثيق مشروع «مسابقة»

آخر تحديث: 2026-07-12  
بيئة الإنتاج: [mosabqah.vercel.app](https://mosabqah.vercel.app)

## 1. نبذة

منصة مسابقات عربية لحظية تعمل على الويب. يدير المقدم الجلسة من لوحة التحكم، وينضم المتسابقون من الجوال برمز غرفة ثابت أو رابط/QR، وتُعرض التجربة العامة على شاشة تلفزيونية أو بروجكتر.

التطبيق يدعم حسابات الإدارة والمقدمين والمتسابقين، مع إمكانية دخول المتسابق كضيف. الواجهة RTL وبنمط بصري داكن.

## 2. التقنية والتشغيل

| البند | القيمة |
| --- | --- |
| الواجهة | Next.js 16 + React 19 + TypeScript |
| الهوية | Firebase Authentication |
| البيانات اللحظية | Cloud Firestore |
| الخادم | Next.js Route Handlers + Firebase Admin SDK |
| النشر | Vercel Production |
| الموقع | `https://mosabqah.vercel.app` |

### أوامر أساسية

```bash
npm run build
vercel --prod --yes
firebase deploy --only firestore:rules --project mosabqah-7cacd
```

لا تُحفظ ملفات مفاتيح Firebase Admin أو `.env.local` داخل Git.

## 3. الأدوار والصلاحيات

### المدير (admin)

- تحكم كامل في المستخدمين، بنك الأسئلة، الجلسات، الأرشيف والتقارير.
- إنشاء حسابات مقدم أو متسابق، تعطيلها، تفعيلها، حذفها وإعادة تعيين كلمة المرور.
- إظهار أو إخفاء أنماط الألعاب عن المقدمين من لوحة الإدارة.
- يرى التقارير العامة وتقارير كل مقدم منفصلة.
- وحده يستطيع تعديل بنك الأسئلة وحذف أرشيف الفائزين.

### المقدم (presenter)

- يحجز رمز غرفة ثابتاً من 4 أرقام.
- ينشئ جلسات أو يحفظها كمسودات ويعود إليها لاحقاً.
- يدير اللاعبين والأسئلة والفرق والنقاط وشاشة التلفزيون.
- يرى أرشيف وتقارير جلساته في الواجهة.

### المتسابق (player)

- يستطيع التسجيل بحساب متسابق أو اللعب كضيف.
- يدخل برمز الغرفة أو رابط مباشر `/player?room=XXXX`.
- يختار لوناً/فريقاً مفضلاً، ويمكن للمقدم إعادة توزيعه بين الفرق.

## 5. بنك الأسئلة

### أنواع الأسئلة

1. **سؤال نصي**: أربعة خيارات إلزامية.
2. **سؤال صورة/علم**: صورة مع ثلاثة خيارات على الأقل. لعبة «كشف الستار» تشترط وجود أربعة خيارات.
3. **الكلمة المفقودة**: الفئة في نص السؤال والكلمة الصحيحة في الخيار الأول.

يظهر نموذج الإضافة عبر تبويبات: سؤال نصي، سؤال صورة، الكلمة المفقودة.

### الإدارة والاستيراد

- بحث وفلترة حسب النوع والتصنيف والصعوبة.
- معاينة مصغرة للصور والأعلام داخل القائمة.
- تعديل وحذف السؤال.
- استيراد Excel/CSV، مع قوالب منفصلة للنصي والصور.
- تنظيف التكرارات المنفذ سابقاً يعتمد على نص السؤال والخيارات والإجابة والتصنيف.
- فلتر **صور وأعلام** يعرض أسئلة الوسائط فقط.

### الأعلام والوسائط

- توجد أصول الأعلام ضمن `public/flags/` وتخدم مباشرة من الموقع لتجنب الاعتماد على Firebase Storage.
- سكربت الاستيراد: `scripts/import-flag-questions.mjs`.
- Firebase Storage غير مفعّل حالياً؛ رفع صور الإدارة إلى Storage يحتاج تفعيل الخدمة وخطة مناسبة. الأعلام الحالية لا تعتمد عليه.
- الصور تُعرض بـ `object-cover` في نمط كشف الستار بنسبة 16:9؛ أي صورة مرفوعة تظهر موحدة بصرياً على شاشة المسرح.

## 6. الجلسات وغرفة التحكم

### دورة الجلسة

1. المقدم ينشئ التحدي من مكتب الألعاب ويختار الأسئلة يدوياً أو عشوائياً.
2. يشارك رمز الغرفة أو QR/رابط واتساب.
3. تظهر غرفة انتظار للمقدم والمتسابقين ثم يغلق المقدم الانضمام عند البدء.
4. التحكم مقسم إلى تبويبات: التحكم، الأسئلة، اللاعبون.
5. عند نهاية الجلسة تؤرشف النتائج ويتوّج الفائزون.

### الأسئلة داخل الجلسة

- زر **سؤال عشوائي** يختار من الأسئلة المتبقية فقط.
- فور طرح السؤال ينتقل إلى قائمة **الأسئلة المستخدمة** ولا يمكن طرحه مرة أخرى.
- تظهر قائمة منفصلة للأسئلة المستخدمة للرجوع إليها دون إمكانية إعادة تشغيلها.
- بعد عرض النتيجة بست ثوانٍ، يعود اللاعبون تلقائياً إلى شاشة انتظار تحفيزية.

### لوحة المراقبة الحية

أثناء السؤال يرى المقدم:

- الوقت المتبقي.
- عدد من أجاب ومن ينتظر.
- قائمة المتسابقين وحالة إجاباتهم.
- أيقونات استخدام حذف إجابتين أو تمديد الوقت.
- حالة الاتصال: متصل، ضعيف، أو غير متصل.
- زر بارز لإنهاء السؤال وكشف الإجابة فوراً.

### الفرق والألوان

- توزيع المتسابقين حسب سعة الفريق/اللون.
- نقاط فردية ونقاط مجمعة لكل فريق.
- نقل متسابق من فريق إلى آخر من لوحة المقدم.
- شاشة التلفزيون تعرض ترتيب الفرق والفائز الجماعي، مع بقاء التتويج الفردي.

### أدوات المقدم الإضافية

- إضافة أو خصم نقاط يدوياً لكل متسابق.
- بث تلميح فوري.
- تخصيص شاشة التلفزيون: شعار، حجم الخط، لون الخلفية، وضع كروما/شفاف.
- QR، نسخ الرابط، ومشاركة واتساب.
- عداد لوقت انطلاق المسابقة.

## 7. أنماط الألعاب

### 7.1 تحدي الأسئلة

- أسئلة نصية أو صور وخيارات.
- 3 ثوانٍ استعداد قبل بدء العداد.
- النقاط تعتمد على الوقت: عند مدة 45 ثانية وإجابة في الثانية 5 يحصل المتسابق على 40 نقطة.
- عند إجابة الجميع تنكشف النتيجة، أو عند انتهاء الوقت تُعد الإجابات المفقودة خاطئة.
- تظهر أفضل ثلاثة بعد الكشف لمدة 6 ثوانٍ.

### 7.2 لعبة الكراسي

- عدد الكراسي يحسب تلقائياً: عدد المتسابقين النشطين ناقص اثنين، بحد أدنى كرسي واحد.
- أول متسابق يحجز كرسيه يتأهل، ويُقصى من لم يحجز.

### 7.3 الزنزانة (Survival)

- الخطأ أو عدم الإجابة يقصي اللاعب من الجولة.
- المتبقيون فقط ينتقلون للسؤال التالي.

### 7.4 حرب الفواكه/الدول (Faction)

- توزيع أحمر/أخضر تلقائي عند بدء اللعب.
- تجمع نقاط الأفراد لصالح الفريق.
- شاشة التلفزيون تعرض شريط القوة بين الفريقين.

### 7.5 أمبوستر

- المقدم يحدد كلمة وفئة، والنظام يختار الإمبوستر.
- مرحلة نقاش ثم تصويت ثم كشف الهوية.

### 7.6 عجلة الروليت

- المقدم يحدد المتسابق الفائز أو يختار المتصدر.
- الفائز يوقف العجلة من جواله، ثم تظهر الجائزة للجميع.

### 7.7 الكلمة المفقودة

- يختار المقدم كلمات من بنك الكلمات ويحدد عدد القلوب: 5 أو 7 أو 10.
- تصل الكلمة للمتسابق بترميز Base64 خفيف لتُعالج الحروف محلياً؛ التحقق النهائي للفوز يتم على الخادم.
- لوحة مفاتيح عربية كاملة، حروف صحيحة خضراء وخاطئة حمراء وغير قابلة للضغط.
- لا يتم استدعاء الخادم عند كل حرف؛ فقط عند كشف الكلمة أو نفاد القلوب.
- شاشة التلفزيون تعرض الكلمة المخفية وشريط عدد من اكتشفوها فعلاً، ثم الكلمة وأسرع ثلاثة عند الكشف.

### 7.8 تخمين الصورة — كشف الستار

- يستخدم أسئلة صور ذات أربعة خيارات.
- إعدادات الشبكة:
  - سهل: 4×4 (16 مربعاً).
  - متوسط: 6×6 (36 مربعاً).
  - صعب: 8×8 (64 مربعاً).
- المقدم يطرح الصورة ثم يضغط **بدء الكشف**.
- تختفي المربعات عشوائياً كل 3 ثوانٍ؛ في الصعب تختفي 3 مربعات كل مرة.
- المتسابق يرى الصورة المغطاة وخياراته، ولا تُفعل الخيارات إلا بعد بدء الكشف.
- شاشة العرض تعرض الصورة المغطاة والخيارات ثم الصورة الكاملة والخيار الصحيح بعد الكشف.
- النقاط تنازلية مع الزمن، لذلك التخمين المبكر قبل انكشاف أجزاء أكثر يعطي نقاطاً أعلى.

## 8. الأرشيف والتقارير

- عند إنهاء الجلسة تحفظ: المقدم، الفائز، نقاطه، عدد المشاركين، وترتيب المشاركين للجلسات الجديدة.
- لوحة الأدمن تعرض تقريراً عاماً لكل المقدمين، أو تقريراً منفصلاً عند اختيار مقدم محدد.
- بطاقات الملخص: عدد المقدمين، الجلسات المنتهية، الأرشيفات، والمشاركون.
- لوحات فائزين مشتقة من الجلسات المؤرشفة للمقدم المختار.
- تصدير تقرير PDF حسب التقرير العام أو المقدم المحدد.
- حذف الأرشيف يحتاج تأكيداً ومتاح للأدمن فقط.

## 9. Firestore: النموذج الأساسي

```text
users/{uid}
questions/{questionId}
sessions/{sessionId}
  players/{playerId}
  answers/{answerId}
winners/{winnerId}
leaderboard/{playerName}
platformSettings/gameModes
```

### حقول جلسة مهمة

- `roomCode`, `createdBy`, `status`, `questionStatus`, `questionIds`.
- `usedQuestionIds`: يمنع إعادة طرح السؤال في الجلسة نفسها.
- `questionStartedAt`, `questionPlayerIds`, `roundWinners`.
- `teamsEnabled`, `teamSize`.
- `wordMaxAttempts` للعبة الكلمة المفقودة.
- `imageRevealGrid`, `imageRevealOrder`, `imageRevealStartedAt` لنمط كشف الستار.

## 11. ملاحظات تشغيلية

- نطاق الإنتاج `mosabqah.vercel.app` يجب أن يبقى ضمن Authorized Domains في Firebase Auth.
- قاعدة Firestore قد تطلب إنشاء Index لبعض الاستعلامات المركبة؛ اتبع الرابط الذي يعرضه Firebase عند ظهور الخطأ.
- لا تستخدم حقولاً بقيمة `undefined` في Firestore. دوال إنشاء/تحديث الجلسات تنظفها تلقائياً.
- يجب اختبار أي نمط جديد بمتصفح للمقدم وآخر للمتسابق وثالث للتلفزيون للتأكد من التزامن.
- `mosabqah_project_context.md` هو أرشيف تقني قديم طويل؛ هذا الملف هو المرجع التشغيلي الحالي والمحدّث.
