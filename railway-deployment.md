# 🚀 دليل نشر تطبيق مسابقة عصومي على منصة Railway.app

يوثق هذا الملف الخطوات التفصيلية لنشر وتشغيل التطبيق على منصة **Railway.app** مع ضمان حفظ قاعدة البيانات بشكل دائم باستخدام **Persistent Volumes**.

---

## 🌟 لماذا منصة Railway؟
- **سرعة استجابة فائقة**: لا ينام السيرفر مطلقاً (بعكس النسخة المجانية لـ Render)، مما يمنع حدوث بطء عند بدء المسابقة.
- **اتصال مستقر**: أداء ممتاز لبروتوكول WebSockets لنقل إجابات اللاعبين بلحظتها.
- **سهولة الإعداد**: ربط مباشر مع مستودع GitHub وبناء تلقائي عند كل تحديث للأكواد.

---

## 🛠️ خطوات النشر خطوة بخطوة

### 1️⃣ ربط المشروع وإنشائه
1. توجه إلى [railway.app](https://railway.app) وسجل الدخول باستخدام حسابك في **GitHub**.
2. اضغط على زر **New Project** (مشروع جديد).
3. اختر **Deploy from GitHub repo** (نشر من مستودع GitHub).
4. حدد مستودع المشروع الخاص بك: `Mosabqah`.

---

### 2️⃣ تكوين متغيرات البيئة (Environment Variables)
في واجهة الخدمة الخاصة بك داخل Railway، انتقل إلى تبويب **Variables** وأضف المتغيرات التالية بدقة:

| المتغير | القيمة | الوصف |
| :--- | :--- | :--- |
| `PORT` | `3005` | منفذ تشغيل الخادم الافتراضي للتطبيق |
| `ADMIN_PASSWORD` | `اكتب_كلمة_المرور_الخاصة_بك` | الرقم السري للدخول للوحة الأدمن للتحكم بالمسابقة |
| `DATA_DIR` | `/data` | المجلد المخصص لحفظ ملف قاعدة البيانات داخل وحدة التخزين الدائمة |

---

### 3️⃣ إعداد وحدة التخزين الدائمة (Persistent Volume) لبيانات الأسئلة
لضمان عدم حذف الأسئلة المضافة وقائمة اللاعبين عند تحديث الموقع أو إعادة تشغيل الحاوية، يجب ربط وحدة تخزين مستمرة:

1. اضغط على بطاقة الخدمة الخاصة بمشروعك في لوحة Railway.
2. توجه لتبويب **Settings** (الإعدادات).
3. انزل لأسفل حتى تصل إلى قسم **Volumes**.
4. اضغط على زر **Add Volume** (إضافة وحدة تخزين).
5. في حقل **Mount Path** (مسار الربط)، اكتب المسار التالي تماماً:
   `/data`
6. احفظ الإعدادات. سيقوم Railway بإعادة تشغيل التطبيق وربط المجلد تلقائياً.

---

## ☁️ خيار بديل: استخدام قاعدة بيانات Turso السحابية
إذا كنت تفضل تخزين الأسئلة سحابياً بالكامل دون استخدام الـ Volumes، يمكنك إنشاء قاعدة بيانات مجانية على منصة **Turso**، ثم إضافة المتغيرين التاليين في تبويب **Variables** بـ Railway:

- `TURSO_URL`: رابط قاعدة البيانات السحابية المقدم من Turso.
- `TURSO_AUTH_TOKEN`: رمز الصلاحية السري الخاص بقاعدة البيانات.

بمجرد إضافة هذه المتغيرات، سيتعرف التطبيق تلقائياً عليها ويقوم بحفظ البيانات سحابياً.
